HTML

  • HTML stands for "Hypertext Markup Language".
  • Elements of HTML are held in tags or brackets (< >) each set of brackets must be closed with a (/).
  • The primary elements are the head and body.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Other elements include !DOCTYPE, title, link, img, meta, style, script, etc.
  • Nesting and ordering are important to using HTML.

CSS

  • CSS stands for Cascading Style Sheet.
  • CSS defines the colors, aesthetics, visual layout, and organization.
  • There are three CSS styles: inline CSS, internal CSS style sheet, and external CSS style sheet. They are listed in increasing amount of detail.
  • An external CSS sheet must be linked after the final meta element and before the title element.
  • CSS properties and their corresponding values are applied in "Declarations".
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout main: moves to main branch
  • git checkout -b feature/branch-name: creates a new branch and switches to it
  • git add -A: stages changes
  • git commit -m "description of changes": commits stated changes
  • git pull origin main: checks that current branch is up to date, necessary before a pull request
  • git push origin feature/branch-name: pushes changes, local main branch should be updated afterward

JavaScript

  • JavaScript adds interactivity to a webpage, one such way it does so is through Application Programming Interfaces (APIs).
  • A variable is a named container that allows us to store data in our code. Variables can contain strings of information as well.
  • Control flow is the order in which a computer executes code in a script.
  • Two ways in which control flow can be interrupted are through conditional statments (if) and loops.
